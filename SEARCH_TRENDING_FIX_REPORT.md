# Search and Trending Links Fix

## الهدف
منع روابط "الأكثر بحثاً" من توليد طلبات `/search?q=...&_rsc=...` تؤدي إلى 502 عندما تكون الكلمات غير موجودة فعليًا في الموقع أو عندما يفشل Backend search مؤقتًا.

## التعديلات

### 1. الأكثر بحثاً لم تعد كلمات عشوائية
تم تغيير مصدر `trendingSearches` في الصفحة الرئيسية ليعتمد على مقالات ومنشورات موجودة فعليًا، مع روابط مباشرة للمحتوى بدل روابط بحث عامة.

النتيجة:
- لا يتم إنشاء رابط بحث لكلمة قد لا تعطي نتائج.
- كل عنصر في "الأكثر بحثاً" يذهب إلى مقال أو منشور موجود.
- تم تعطيل `prefetch` لهذه الروابط لتجنب طلبات RSC الخلفية غير الضرورية.

### 2. Search API أصبح آمنًا
تم تعديل `src/app/api/search/route.ts` بحيث لا يرجع 502 أو 500 للمتصفح عند فشل Backend API.

بدل ذلك يرجع:
```json
{
  "success": true,
  "results": [],
  "warning": "تعذر تحميل نتائج البحث حاليًا."
}
```

### 3. صفحة البحث أصبحت تتحمل فشل API
تم تعديل `src/app/(main)/search/page.tsx` حتى لا تكسر الصفحة عند رجوع non-JSON أو status غير ناجح من `/api/search`.

### 4. تعطيل prefetch لروابط البحث الثقيلة
تم إضافة `prefetch={false}` لروابط البحث داخل صفحة البحث وبعض روابط `/search` العامة.

## الأثر المتوقع
- توقف أخطاء `GET /search?q=...&_rsc=... 502` الناتجة عن الأكثر بحثاً.
- تحسين استقرار الصفحة الرئيسية.
- منع جلب اقتراحات غير مضمونة أو غير موجودة.
- إبقاء صفحة البحث تعمل برسالة واضحة بدل خطأ Gateway.
