# Content Audit Page Split Report

تم فصل صفحة فحص المحتوى إلى صفحات تشغيلية مستقلة لتقليل الزخم داخل `/dashboard/content-audit` ورفع قابلية الصيانة وتجربة الاستخدام.

## المسارات النهائية

- `/dashboard/content-audit`: فحص المحتوى والسياسات، تقارير AdSense، التصدير، وتحليل نتيجة محددة.
- `/dashboard/content-quality`: مركز تحسين جودة المحتوى بالذكاء الاصطناعي وتشغيل Batch Jobs.
- `/dashboard/content-review`: قائمة المراجعة البشرية واعتماد/رفض معاينات التحسين.
- `/dashboard/ai-costs`: متابعة تكلفة الموديلات، التوكنات، الطلبات، ونسبة النجاح والفشل.

## ما تم تغييره

- إزالة `ContentQualityBatchPanel` من صفحة فحص السياسات.
- إزالة `ContentAIReviewQueuePanel` من صفحة فحص السياسات.
- إزالة `ContentAIModelCostPanel` من صفحة فحص السياسات.
- إنشاء صفحات مستقلة للمكونات الثلاثة.
- تحديث Sidebar بروابط تشغيلية واضحة لكل وظيفة.

## ملاحظات تشغيل

لم يتم تغيير مسارات API، لذلك لا يحتاج الباك اند إلى تعديل في هذه المرحلة.
